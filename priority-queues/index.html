Priority Queues

When there's too much to do and too little time, how do we choose what to do next?

We choose whatever has the highest priority.

Priority queues are data structures which might not be sorted, but still give us easy access to the highest priority item. This means we don't have to keep a sorted list, which is expensive to maintain.

Priority queues have two main methods:

  1. enqueue: O(log n), add an item with a priority
  2. dequeue: O(log n), remove the highest priority item

They may include additional methods:

  1. isEmpty: O(1), returns true if no items are in the priority queue
  2. peek: O(1), returns the item with highest priority without removing it

Implementation

Priority queues are usually implemented with heaps.

let PriorityQueue = (things) => {
  let heap = makeHeap(things),
      { size, items } = heap
  
  let enqueue = (item) => heapInsert(item, heap)
  let dequeue = ()     => extractMin(heap)
  let isEmpty = ()     => size === 0
  let peek    = ()     => items[1]

  return {
    enqueue,
    dequeue,
    isEmpty,
    peek
  }
}

let items = [5,4,3,2,1]
let priorityQueue = PriorityQueue(items)
